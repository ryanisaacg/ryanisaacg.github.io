What I got done on Felt in 2024

20 Jan 2024

Over the past year-and-change I’ve been working on a hobby programming language called Felt[1]. Felt is a language designed for arts and crafts: it’s for small programs (either practical or fun) that don’t need industrial strength. I last wrote about it during my batch at the Recurse Center, and that blog post remains a good overview of the general vibe and goals of the project.

I think of Felt as bringing a systems-programming mindset to a high-level language. I want to take the core innovation of Rust (state may either be mutable or aliased but never both) and package it into a simpler language that’s easier to learn, write, and iterate on. The prototypical Felt program is a web game or toy[2], something where memory locality matters but a reference counter won’t be performance-crushing.

What’s There

  • A non-optimizing, whole-program compiler which outputs WASM[3]
    • There’s a basic type-checker (though the type system is also basic so it’s fine)
    • There’s a borrow-checker, which is currently Good Enough™️. Borrows are second-class and limited to the scope they’re created, so checking them isn’t too hard
    • Polymorphism is currently provided through Go-like interfaces, available on both structs and unions
  • A basic LSP (that doesn’t understand imports at all, which is a major stumbling point)
  • A treesitter grammar

What’s Planned

  • Generics (either monomorphic or not). This is a big goal for 2025
  • Containers. There are growable arrays and reference counted boxes provided as compiler built-ins. There are also dictionaries, but they only support integer keys because I’m not sure how to allow custom types without a ton of compiler machinery. My plan is to add enough to the language that containers can be implemented as part of a standard library, which will solve this problem
  • Packages
  • Function references
  • Iteration
  • Coroutines

  1. It’s gone through some identity crises, but for now it’s going by Felt. Previously I was calling it Brick, but that didn’t convey the vibe I was going for. ↩︎

  2. Which is a big reason why I compile to WASM ↩︎

  3. Much of the compiler is written from scratch without consulting prior art. This is part of the fun for me, but I’m sure it means I’ve made suboptimal decisions. ↩︎
