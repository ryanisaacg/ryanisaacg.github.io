Let's work our way out from the editor to the operating system: I use neovim. It's like vim, but neo! Killer features include built-in lua configuration and scripting support, async and terminal embedding (since upstreamed to Vim 8, is my understanding), and built-in support for the Language Server Protocol. LSP is the really killer tool here; it gives me all of the niceties of an IDE without ever leaving my terminal. TODO: extensions

Out one layer from the editor is the shell. Personally I run fish shell and have for years, after a fair stint with zsh. Fish has sensible and pleasant defaults out of the box, and is a fair bit nicer to script in than its more standard cousins like bash. These days, a shell doesn't feel complete to me without fzf (for finding files quickly) and rg (for finding contents quickly). That's about all there is to it!

Very rarely am I doing serious work in a shell without that shell sitting in tmux. This is one piece of my setup that feels the least critical; any decent terminal can do splits and tabs on its own nowadays. I picked up the habit while ssh'ing, though, and all the keyboard shortcuts feel like second nature. Plus it persists my sessions when I accidentally close a terminal tab, which is a nice bonus.

Lastly, we get to the terminal emulator. My preferred terminal is iTerm 2; its UI just feels slicker and more put-together than other options. This is one of the only reasons I prefer macOS to Linux, if I'm being entirely honest.
